# Settings 配置中心导入指南

本文用于统一配置中心（`/settings`）中的 JSON 导入操作说明，重点覆盖：

- 导入模式选择
- 冲突明细含义
- 常见失败原因与排查

## 1. 导入前检查清单

1. 先确认当前是目标策略画像（避免覆盖错对象）。
2. 导入前先点一次“导出配置 JSON”或“导出画像仓库”做备份。
3. 若使用覆盖模式，请先在预览区核对“覆盖目标”列表。
4. 若显示“超出上限将截断”，先清理历史画像再导入。

## 2. 导入模式如何选

### 新增（append）

- 适合场景：保留现有画像，新增一批测试/实验画像。
- 行为：同 ID 会自动改名（例如 `exam`, `exam_2`, `exam_3`）。
- 风险：画像数量会快速增长，可能触发上限截断。

### 覆盖（overwrite）

- 适合场景：你明确要替换线上/演示正在用的固定画像 ID。
- 行为：同 ID 直接覆盖；同批次重复同 ID 时，后面的条目生效（last wins）。
- 保护：必须勾选“确认覆盖已存在画像”才能执行。

## 2.5 二次确认弹层

点击“导入为策略画像”或“导入画像仓库”后，会先弹出确认框，展示：

- 本次总导入条数 / 预计新增 / 预计覆盖 / 实际写入条数
- 覆盖目标摘要（若存在）
- 截断风险（超上限）
- 同批重复处理说明（自动改名或后者生效）

确认后才会真正写入本地仓库。

## 3. 冲突明细表字段解释

当检测到重复 ID 时，可展开“冲突明细”查看：

- `序号`：导入包中的原始顺序（从 1 开始）
- `原始 ID`：JSON 内声明的 ID
- `目标 ID`：最终写入本地仓库的 ID
- `处理`：`新增` 或 `覆盖`
- `策略`：
  - `新增自动改名`：append 模式，自动分配新 ID
  - `覆盖后者生效`：overwrite 模式，同 ID 后项覆盖前项

冲突明细支持：

- 按处理类型筛选（全部/仅新增/仅覆盖）
- 按序号/原始 ID/目标 ID 关键词搜索
- 导出当前筛选结果为 CSV（便于评审与留档）

## 4. 常见问题与排查

### 提示“JSON 格式错误，无法解析”

- 检查是否缺少逗号、引号或多余注释。
- 先用在线 JSON 校验器格式化，再粘贴回配置中心。

### 提示“未识别为配置包或画像包”

- 配置包需包含 `dashboard/workspace/kb`。
- 单画像包推荐结构：`{ "profile": { ... } }`。
- 画像仓库包推荐结构：`{ "store": { "profiles": [...] } }`。

### 提示“可导入画像数量为 0”

- 通常是因为画像上限已满，或导入条目无有效 `bundle`。
- 先删除不需要的画像，再重试。

### 覆盖后结果与预期不一致

- 查看冲突明细中的 `序号` 和 `策略`，确认是否发生了“后者覆盖前者”。
- 建议在覆盖前先导出当前仓库快照，必要时可回滚。

## 5. 推荐操作流程（生产/演示）

1. 导出当前仓库备份。
2. 粘贴 JSON，观察预览（新增/覆盖/截断/重复组）。
3. 如需覆盖，勾选确认并展开冲突明细逐条核对。
4. 执行导入后，确认底部提示并切换画像抽查。
5. 导入成功后再导出一次，形成“变更后快照”。

> 提示：在二次确认弹层中，如果你原本选择了覆盖模式，也可以点击“仅导入新增项”快速改为新增导入，再确认执行。

## 6. 导入日志与回滚

配置中心会记录最近导入操作（最多 12 条），每条包含：

- 导入来源（单画像/画像仓库）
- 导入模式（新增/覆盖）
- 新增数、覆盖数、截断数、重复冲突数

支持操作：

- 回滚到导入前：一键恢复该次导入前的画像仓库与活动配置。
- 回滚影响预览：在执行回滚前可看到画像“新增/删除/变更”数量与参数差异项数。
- 回滚确认弹层：点击“查看并回滚”会先展示逐项差异，再二次确认执行。
- 回滚差异筛选与导出：支持按“新增回补/移除恢复/画像更新”筛选，支持关键词检索并导出当前筛选结果 CSV。
- 弹层快捷操作：支持 `Esc` 关闭导入确认或回滚确认弹层。
- 按来源/模式/关键词/时间区间筛选日志，定位目标导入动作。
- 快速区间筛选：支持“近 1 天 / 近 7 天”一键设置起止日期。
- 导出筛选 JSON：导出当前筛选结果用于留档。
- 导出单条 JSON：对某条日志单独导出，便于精细追踪。
- 从 JSON 草稿导入：可将历史导出日志重新导回本地（自动去重并按上限截断）。
- 清空日志：删除本地导入日志记录，不影响当前策略配置。
